## 🚨 핵심 발견 (가장 중요)

### IRGO IR자료실 ≠ "IR 담당자 정보를 담은 자료"

**예상 vs 현실:**
- **예상**: IR북에는 회사 IR 담당자 이름·전화·이메일이 표지/마지막 페이지에 있음
- **현실**: IRGO에 올라온 자료는 **거의 100% 분기 실적 발표 자료**
  - 예: "두산 두산, 2026년 1분기 경영실적 발표"
  - 예: "LG전자 LG전자, 2026년 1분기 경영실적 발표"
  - 내용: 재무 데이터 + 사업 현황 + Disclaimer
  - **IR 담당자 연락처는 거의 포함되지 않음**

**검증한 PDF 샘플 분석:**
- 대한전선 IR자료: 18페이지, 첫 페이지 = 표지, 마지막 페이지 = Appendix(재무표)
- KG스틸 IR자료: 13페이지, 마지막 페이지 = 별도 재무상태표
- 동아쏘시오홀딩스: 12페이지, 마지막 페이지 = (빈 페이지)

→ IR 담당자 contact 정보는 거의 없음

### 정규식 거짓 매칭 문제

추출 시도된 6건이 모두 **disclaimer 텍스트의 거짓 매칭**:
- "0100-1400-1050" — 실제로는 발표일자 (2025.04.29)
- "023-2024-2025" — 연도 패턴 (2023, 2024, 2025)
- "한가닥 책임" / "굳 책임" / "독립된 책임" — disclaimer 동사구
- 모두 폐기 처리

→ 분기 실적 자료에서 contact 추출은 노이즈만 생성

## 🛠 작업 진행 상세

### 1단계: 라이브러리 설치 ✅
- `pdfplumber 0.11.9` (PDF 텍스트 추출)
- `python-pptx 1.0.2` (PPTX)
- `python-docx` (DOCX)
- `playwright` (KIND용 — 미사용)

### 2단계: IRGO 구조 분석 ✅
- URL 패턴: `https://m.irgo.co.kr/IR자료/{id}/TB/{회사명}-{타이틀}`
- PDF 다운로드: `https://file.irgo.co.kr/data/BOARD/ATTACH_PDF/{HASH}.pdf`
- 페이지네이션: 무한스크롤 (직접 ID 순회만 가능)
- ID 73710 (최신) → 30000 정도까지 범위

### 3단계: KIND 구조 분석 ⚠️
- IR자료실 URL: `kind.krx.co.kr/corpgeneral/irschedule.do?...&gubun=iRMaterials`
- 검색 폼 + AJAX 결과 로드 — 단순 HTTP 요청으로는 결과 못 가져옴
- → **Playwright/Selenium 필요** (시간 미확보로 미시도)

### 4단계: IRGO 수집 스크립트 ✅
- `scripts/collect_irgo_irbooks.py` — ID 순회 + 회사 매칭 + PDF 다운로드
- 차단 방지: User-Agent 회전 + 4~5초 랜덤 딜레이
- 메타데이터 영속화 (재실행 시 이어서)
- 최종 130개 ID 시도 → 81개 PDF 다운로드

### 5단계: 파싱 스크립트 ✅
- `scripts/parse_ir_books.py` — PDF/PPTX/DOCX 텍스트 추출 + contact 정규식
- 첫 2페이지 + 마지막 2페이지에 집중
- v5 정규식 재활용 (이메일/전화/이름/직책)

### 6단계: 추출 결과 분석 ⚠️
- 81개 PDF 파싱 → 6건 추출 → **모두 거짓 매칭**
- 유효 contact: **0건**
- DB에 저장된 6건 모두 삭제 (cleanup)

## ✅ 잘된 것

1. **IRGO 사이트 구조 파악** — URL 패턴, PDF 직접 링크, 회사명 추출 로직
2. **차단 없이 안전한 크롤링** — 130개 ID 시도하며 차단 0회
3. **회사 매칭 87%** — 검색 페이지에서 회사명 → corp_code 매칭 정확
4. **PDF 파싱 인프라 구축** — PDF/PPTX/DOCX 모두 처리 가능 (재사용 가능)
5. **디스크 절약** — 회사당 1개만 다운로드 (메타 추적)
6. **거짓 매칭 자동 정리** — 노이즈 데이터 DB에 영구 저장 안 됨

## ❌ 잘 안 된 것

1. **IRGO 자료 성격 미스매치** — 분기 실적 자료에는 IR 담당자 정보 거의 없음
2. **disclaimer 텍스트 노이즈** — 정규식이 동사구, 연도 등을 잘못 추출
3. **KIND 접근 실패** — JS 렌더링 필요로 단순 HTTP 안 됨
4. **추출률 0%** (유효 contact 기준) — 81개 PDF에서 단 한 건도 유효 정보 없음

## 🎓 배운 교훈

1. **데이터 소스 사전 검증 필수**: "IR북" 단어가 의미하는 자료 형태가 사이트마다 다름
2. **IRGO IR자료실은 IR 담당자 contact용 X**: 투자자에게 실적/전망 알리는 자료 위주
3. **정규식 거짓 매칭 위험**: 도메인 제한 없이 추출하면 disclaimer 등에서 잘못 매칭
4. **사용 케이스에 맞는 데이터 소스 선택**: 회사 홈페이지 IR 페이지 vs 분기 발표 자료는 다른 목적
